08 · Frozen & Responsive Columns

Two complementary ways to handle tables with more columns than will fit. They don't mix in one table (responsive collapse removes the overflow columns, so there's nothing left to scroll to), so each is shown on its own below. ← back to index

Frozen, movable & resizable

This table is wider than the page, so it scrolls horizontally (scroll the table sideways →). The Name column is frozen to the left and the actions column is frozen to the right, so both stay visible while the middle scrolls. Drag a header to reorder columns, or drag a column border to resize.

Responsive collapse

Same data, but here the columns fit the width and the lowest-priority ones (responsive value highest) collapse into an expandable row as space runs out (narrow your browser window →). Click the + on a row to reveal the hidden fields.
